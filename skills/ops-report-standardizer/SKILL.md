# 运维巡检报告标准技能

## 目标

基于已有证据生成统一、可维护、适合生产环境的中文巡检报告。

## 输出要求

- 全部使用简体中文
- 必须包含执行摘要
- 必须包含健康分
- 必须包含 `Critical` / `Warning` / `Info`
- 必须包含证据表
- 必须包含建议动作和建议责任方
- 必须包含不确定性说明

## 约束

- 不允许把没有证据支持的猜测写成结论
- 如果证据不足，明确写“不足以下结论”
- 如果是 WSL、容器、测试环境，要把背景写清楚
- 优先指出真正影响生产风险的项，不要堆低价值噪声

## 风格

- 结论先行
- 语言简洁
- 可审计
- 可直接给运维或平台负责人阅读
